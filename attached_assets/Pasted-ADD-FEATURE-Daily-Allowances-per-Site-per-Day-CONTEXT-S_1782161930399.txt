ADD FEATURE: Daily Allowances per Site per Day

CONTEXT:
Supervisors sometimes give labours extra cash for travel 
(petrol), lunch, breakfast, tea when site is far away.
These are daily extras on top of base wage.
They apply to ALL present labours at a site on that day.

═══════════════════════════════════════════════════
FIRESTORE CHANGES
═══════════════════════════════════════════════════

Update attendance/{date}/records/{labourId} document:
ADD these fields:
  allowances: {
    petrol: number (default 0),
    lunch: number (default 0),
    breakfast: number (default 0),
    tea: number (default 0)
  }
  totalAllowance: number (default 0)
  grandTotal: number = wageAtTime + totalAllowance

Create new collection: siteAllowances/{date}_{siteId}
  - date: string
  - siteId: string
  - siteName: string
  - setBy: uid
  - setAt: timestamp
  - allowances: {
      petrol: number,
      lunch: number,
      breakfast: number,
      tea: number
    }
  - appliedToCount: number (how many labours got this)

This collection stores what was set site-wide so admin 
can audit it separately.

ALLOWANCE TEMPLATES (Admin Web — Sites page)
═══════════════════════════════════════════════════

On the Sites page, each site has an "Allowances" section:
  Default petrol: ₹ [input]
  Default lunch:  ₹ [input]  
  Default breakfast: ₹ [input]
  Default tea:    ₹ [input]

These are saved on the site document:
sites/{siteId}
  ADD: defaultAllowances: {
    petrol: 200,
    lunch: 0,
    breakfast: 0,
    tea: 0
  }

Flutter reads these defaults when opening the 
allowance bottom sheet — pre-fills the amounts.
Supervisor just toggles ON/OFF and adjusts if needed.

═══════════════════════════════════════════════════
BUGS TO PREVENT
═══════════════════════════════════════════════════

1. If supervisor taps "Skip for today" — all allowance 
   fields stay 0, do not null. Null breaks sum calculations.

2. grandTotal must always = wageAtTime + totalAllowance
   Recalculate on every allowance edit, never trust 
   stale cached value.

3. If no labours are present at a site, do not show 
   the allowance bottom sheet at all.

4. Allowance edits after midnight should require 
   admin PIN / flag as late edit in audit log.

5. Excel export: format all currency columns as 
   numbers not strings so Excel SUM formula works.
