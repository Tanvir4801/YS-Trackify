# YS Trackify – Labour Management App

Flutter-based labour attendance and payment tracking app. Originally built for Android/iOS; configured here to run as a Flutter Web build inside the Replit preview.

## Tech Stack

- **Framework:** Flutter 3.32 (Dart 3.8)
- **State management:** Riverpod + Provider
- **Local storage:** Hive (browser IndexedDB on web)
- **Backend:** Firebase (Auth, Firestore, Cloud Functions) – project `ys-construction`
- **Mobile-only packages** (no-op or limited on web): `mobile_scanner`, `vibration`, `permission_handler`, `flutter_local_notifications`, `audioplayers`

## Project Layout

- `lib/` – Dart source (models, providers, screens, services, widgets)
- `lib/firebase_options.dart` – Firebase config (android, ios, web)
- `web/` – Flutter web bootstrap (auto-generated by `flutter create . --platforms=web`)
- `android/`, `ios/`, `macos/` – Native runners (not used in Replit)
- `functions/` – Firebase Cloud Functions
- `build/web/` – Compiled web output served by `serve.js`
- `serve.js` – Tiny Node static server with SPA fallback (port 5000)

## Replit Setup

- Flutter SDK installed via Nix (`flutter` system dependency).
- Web platform was added with `flutter create . --platforms=web --project-name=ys_trackify`.
- A `web` Firebase target was added to `lib/firebase_options.dart` so `Firebase.initializeApp` works in the browser.
- The app is built once with `flutter build web --release` and served as static assets via `node serve.js` on `0.0.0.0:5000`.

### Workflow

- **Start application** → `node serve.js` (port 5000, webview)

### Rebuilding after Dart changes

The serve workflow only serves the previously compiled `build/web/`. After editing Dart code, rebuild and restart:

```bash
flutter build web --release
```

Then restart the **Start application** workflow.

## Deployment

Configured for **Static** publishing:

- **Build:** `flutter build web --release`
- **Public dir:** `build/web`

## Notes / Limitations on Web

- Some plugins (mobile scanner, vibration, native notifications) are mobile-only and will be inactive in the browser.
- Hive runs on IndexedDB on web – data is per-browser.
- Firebase web config uses a placeholder `appId` reused from the Android config; for full production web auth you should register a dedicated Web app in the Firebase console and replace the values in `lib/firebase_options.dart` (`web` block).
